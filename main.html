How the system works

Arduino reads MQ-2, MQ-6 and MQ-135 gas sensors + DHT11 every 500ms. MQ-2 (A3) detects smoke and LPG; MQ-6 (A0) targets LPG/propane; MQ-135 (A1) monitors air quality. If gas exceeds threshold, it instantly triggers buzzers, fan, closes servo valve, and turns on red LED. Every 2 seconds, Arduino sends a JSON string to ESP32 via SoftwareSerial (D5→GPIO16). ESP32 connects to WiFi and posts this data to your Render server every 3 seconds. The server broadcasts it to this dashboard via Socket.io in real time. Telegram alerts fire on state changes.

  • Safe (green) — All readings normal. No action needed.
  • Warning (amber) — Gas levels rising (80% of threshold). Fan activates, alarm sounds.
  • Danger (red) — Gas exceeds threshold. Valve closes, buzzer ON, fan ON, Telegram alert sent.

System returns to Safe automatically when readings drop. Gas valve will NOT re-open automatically — press Open in Remote Control after verifying area is safe.

  • Do NOT use electrical switches — could ignite gas
  • Open windows and doors immediately
  • Evacuate the area
  • Call gas emergency services
  • Only re-open regulator after area is fully ventilated and safe
  • Open Serial Monitor at 115200 baud — you will see WiFi status and HTTP response codes
  • HTTP -1 = timeout. Render free tier sleeps — first request may take 30–50 s to wake
  • Check that SERVER_URL in ESP32 code exactly matches the URL in Setup tab
  • Check Arduino D4/D5 wires are connected to ESP32 GPIO16/17 (not D0/D1)
  • Disconnect Arduino TX/RX wires before flashing ESP32, reconnect after
  • Go to Setup tab and click Connect to re-link dashboard to server
